## 프록시 패턴은 무엇인가요?

프록시는 **대리인**이라는 뜻입니다. 대상 객체(Real Subject)에 직접 접근하는 대신, 중간에 **프록시 객체**를 두어 접근을 제어하거나 부가적인 기능을 추가하는 디자인 패턴입니다.

대상 객체의 코드를 수정하지 않고도 다음과 같은 기능을 수행할 수 있습니다.
- **보안 (Access Control):** 권한이 있는 사용자만 대상 객체에 접근하도록 제한합니다.
- **캐싱 (Caching):** 이전에 계산하거나 가져온 데이터를 프록시가 저장해 두었다가 서버 부하를 줄이며 빠르게 반환합니다.
- **데이터 검증 (Validation):** 전달된 인잣값이 유효한지 미리 확인합니다.
- **지연 로딩 (Lazy Initialization):** 리소스가 큰 객체는 실제 사용되는 시점에 생성하여 초기 구동 속도를 높입니다.
- **로깅 (Logging):** 객체의 메서드가 호출될 때마다 기록을 남깁니다.

인프라 관점의 예시는 [Nginx](Nginx.md)가 있습니다. Nginx를 클라이언트와 백엔드 사이에서 리버스 프록시 역할로 사용할 수 있습니다. 클라이언트는 서버의 실제 주소를 모르고, Nginx와 통신하고, Nginx는 전달받은 요청을 서버에게 전달하여 통신을 중계할 수 있습니다. 이를 통해 서버 IP를 숨기고, 정적 콘텐츠를 캐싱하여 전달할 수 있으며 여러 서버로 요청을 배분할 수도 있습니다.
